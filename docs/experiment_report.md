# 实验报告

## 1. 实验目的

本实验比较纯文本协作模式与结构化协议协作模式在多 Agent 连续任务中的通信开销、任务耗时、非文本状态传递和共享记忆复用效果。

## 2. 实验任务

实验使用 `tasks/continuous_tasks.json` 中的 10 轮任务：

- A1-A5：openEuler Python Web 服务部署链。
- B1-B5：CSV 数据分析链。

两组任务均包含规划、检索、执行、总结和记忆写入过程，且后续任务可复用前序任务中的执行策略、报告模板和排障经验。

## 3. 指标

| 指标 | 含义 |
|---|---|
| Agent 消息数 | Agent 间消息总数 |
| 文本字符数 | 纯文本消息字符数或协议 JSON 字符数 |
| 估算 token | 按字符数 / 1.8 估算 |
| 状态传递次数 | `state_refs` 被传递的次数 |
| embedding 数据量 | 传递状态对应的字节规模 |
| 总耗时 | 完成全部任务的耗时 |
| 记忆命中率 | 命中共享记忆的任务比例 |
| 检索次数 | 本地语料检索次数 |

## 4. 运行方式

```bash
python scripts/run_benchmark.py --mode both --rounds 10
```

## 5. 结果

运行后查看：

```text
outputs/benchmark_report.md
outputs/benchmark_summary.json
```

报告由系统自动生成，避免手动填写导致不可复现。

## 6. 分析口径

结构化协议模式的优势来自三个方面：

1. 协议消息只传动作、参数、结果和引用，减少自然语言上下文重复。
2. embedding 存在状态池中，通过 `state_ref` 交换，减少文本编解码。
3. 共享记忆命中后，后续任务可复用历史策略并减少本地检索。
